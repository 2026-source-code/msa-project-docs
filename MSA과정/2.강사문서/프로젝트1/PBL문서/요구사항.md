## 프로젝트 최소 요구사항

| 번호 | 최소 요구사항 | 기준 |
|-----|-------------|------|
| 1 | REST API | 최소 5개 이상 |
| 2 | DB 테이블 | 4개 이상 (JPA 연관관계 매핑 포함) |
| 3 | 인증/인가 | 회원가입, 로그인, 역할 구분 (최소 2개 역할) |
| 4 | Redis 세션 관리 | 세션 저장소를 Redis로 구성 |
| 5 | Mustache SSR | Server Side Rendering 화면 구성 |
| 6 | 예외처리 및 유효성 검증 | 글로벌 예외 핸들러 + 서버 사이드 검증 |
| 7 | 페이징/정렬 | 목록 조회 시 페이징 기능 |
| 8 | Git Flow | 브랜치 전략 수립, 스텝 완료 시 master 커밋 |
| 9 | README | 아키텍처 다이어그램 포함 |
| 10 | Docker 배포 (선택) | Docker 기반 배포 환경 구성 |
| 11 | Swagger 문서화 (선택) | API 문서 자동화 |

## 추가 학습 권장 항목 (PBL 자기주도)

| 항목 | 설명 | 기술 키워드 |
|-----|------|-----------|
| 파일 업로드 | 이미지 또는 파일 업로드 기능 | MultipartFile |
| 실시간 알림 | SSE 기반 실시간 알림 | Spring SseEmitter |
| Spring Security | 인증/인가 프레임워크 적용 | SecurityFilterChain |
