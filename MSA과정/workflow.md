# MSA과정 워크플로우

## 핵심 개념: 단일 진입점

```
작성은 각자 편한 곳에서, 확인은 한 곳에서!

┌─────────────────────────────────────────────────────┐
│              홈페이지 대시보드 (단일 진입점)              │
│                                                     │
│  국가 심사관 / 강사가 이 화면 하나로 전부 확인            │
│                                                     │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ │
│  │ 강사 서류     │ │ 학생 서류     │ │ 소스코드      │ │
│  │ (사이트 내부) │ │ (노션 링크)   │ │ (GitHub 링크) │ │
│  │              │ │              │ │              │ │
│  │ PDF 보기     │ │ 클릭→노션이동  │ │ 클릭→GitHub  │ │
│  │ PDF 다운로드  │ │              │ │              │ │
│  └──────────────┘ └──────────────┘ └──────────────┘ │
└─────────────────────────────────────────────────────┘
         ↑                  ↑                ↑
    강사가 입력         학생이 작성        학생이 코드작업
   (홈페이지에서)       (노션에서)         (GitHub에서)
```

### 작동 방식

1. **팀 등록 시** 노션 URL + GitHub URL을 홈페이지에 입력
2. **대시보드**에서 15종 서류를 한 화면에 표시
3. 강사 서류 → 사이트 내부 PDF 직접 보기/다운로드
4. 학생 서류 → 노션 링크 클릭하면 바로 이동
5. 소스코드 → GitHub 링크 클릭하면 바로 이동

```

강사가 입력 → 서류 자동생성   학생이 작성              학생이 코드 관리
+ 대시보드 (단일 진입점)     + 팀 노션 URL 등록       + 팀 GitHub URL 등록
━━━━━━━━━━━━━━━━━━━━━━    ━━━━━━━━━━━━━━━━━━    ━━━━━━━━━━━━━━━━

[평가]                     [프로젝트 문서]           [소스코드]
- 포트폴리오 평가            - 요구사항정의서          - 팀 레포지토리
- 객관식/서술형 평가          - 스프린트계획서          - feature 브랜치

                           - 하브루타노트
[프로젝트 강사 서류]         - 트러블슈팅             [코드리뷰]
- 관찰일지 → PDF            - 팀블로그관리            - master 커밋에 대해
- 평가표 → PDF              - 동료평가                 강사가 리뷰
- 멘토링일지 → PDF           - 프로젝트회고
                           - 코드리뷰기록
[대시보드]                  - 최종발표자료
- 15종 서류 현황 한눈에
- 노션 링크 / GitHub 링크
- PDF 일괄 다운로드
```

### 핵심 원칙

- **강사** = 홈페이지에서만 작업 → PDF 자동 다운로드
- **학생** = 노션에서만 문서 작성 (팀 노션 URL은 홈페이지에 등록)
- **코드** = GitHub (팀 GitHub URL은 홈페이지에 등록)
- **확인** = 홈페이지 대시보드 하나로 전부 확인 (링크 기반)

## 사용 시나리오

### 강사 시나리오

```
프로젝트 시작 전
│
├─ [홈페이지] 과정/교과목/학생/팀 등록
│   └─ 팀 등록 시 노션 URL, GitHub URL 입력
├─ [홈페이지] 시험지 등록 (객관식/서술형/PBL)
│
프로젝트 진행 중 (4주)
│
├─ 매일
│   ├─ 스탠드업 참관
│   ├─ [대시보드 → 노션 링크] 학생 회의록, 하브루타 근거자료 확인
│   └─ [대시보드 → GitHub 링크] 커밋 현황 확인
│
├─ 스텝 완료시
│   └─ [GitHub] master 커밋 코드리뷰 (노션에 리뷰 기록은 학생이)
│
├─ 주 1회
│   └─ [홈페이지] 관찰일지 입력
│       (학생별 대인관계/문제해결 점수 + 근거자료 체크)
│
├─ 멘토링 후 (4회)
│   └─ [홈페이지] 멘토링일지 입력
│       (질문/답변, 피드백, 액션아이템)
│
├─ 중간 발표
│   ├─ [홈페이지] 프로젝트 평가표 입력 (6영역 100점)
│   └─ [홈페이지] 객관식/포트폴리오 평가 진행
│
└─ 최종 발표 / 국가 제출
    ├─ [홈페이지] 프로젝트 평가표 입력
    ├─ [홈페이지] 객관식/포트폴리오 평가 진행
    └─ [대시보드] 15종 서류 한눈에 확인 → PDF 일괄 다운로드
```

### 학생 시나리오

```
프로젝트 시작
│
├─ 1주차
│   ├─ [노션] 요구사항정의서 작성 (MVP, DoD, ERD, API)
│   ├─ [노션] 스프린트계획서 작성
│   └─ [GitHub] 레포 생성, 브랜치 전략 설정
│
├─ 매일
│   ├─ [노션] 회의록 작성 + 회의 사진 첨부
│   ├─ [노션] 하브루타노트 작성 (개인)
│   └─ [GitHub] feature 브랜치에서 코드 작업
│
├─ 스텝 완료시 (회원가입, CRUD, 핵심기능 등)
│   ├─ [GitHub] master에 커밋 → 강사 코드리뷰 받음
│   └─ [노션] 코드리뷰 결과 기록
│
├─ 에러 발생시
│   └─ [노션] 트러블슈팅 기록 (증상→원인→해결→배운점)
│
├─ 수시
│   └─ [노션] 블로그 작성 → 노션 복제, 주 1회 발표
│
├─ 스프린트마다
│   └─ [노션] 스프린트계획서 + KPT 회고
│
├─ 중간/최종
│   ├─ [노션] 동료평가 (개인)
│   └─ [홈페이지] 시험 응시 (객관식/포트폴리오)
│
└─ 종료
    └─ [노션] 프로젝트회고 작성 + 모든 문서 정리
```

### 국가 심사관 시나리오

```
홈페이지 접속 → 과정 선택 → 대시보드
│
├─ 팀 목록에서 전체 진행률 한눈에 확인
│
├─ 팀 클릭 → 15종 서류 상세 화면
│   ├─ 강사 서류 → [PDF 보기] 클릭 → 바로 확인
│   ├─ 학생 서류 → [노션 열기] 클릭 → 노션 페이지로 이동
│   └─ 소스코드 → [GitHub 열기] 클릭 → GitHub 저장소로 이동
│
└─ [PDF 일괄 다운로드] → 강사 서류 전체 ZIP 파일
```

## 코드리뷰 방식

### 변경 전 (비현실적)
```
모든 PR마다 → 팀원 1명 리뷰 → 노션에 기록
→ PR이 하루에도 여러 개, 강사가 전부 확인 불가
```

### 변경 후 (실제 운영)
```
학생: feature 브랜치에서 자유롭게 작업
      ↓
      특정 스텝 완료 (예: 회원가입, 핵심 CRUD, Redis 등)
      ↓
      master에 커밋
      ↓
강사: master 커밋만 코드리뷰
      ↓
학생: 리뷰 결과를 노션 코드리뷰 문서에 기록
```

### 코드리뷰 대상 스텝 (예시)

| 스텝 | 시기 | 리뷰 포인트 |
|------|------|-----------|
| 프로젝트 초기 설정 | 1주차 | 패키지 구조, 설정 파일 |
| 회원 관리 | 1~2주차 | 엔티티 설계, 인증 로직 |
| 핵심 도메인 CRUD | 2주차 | JPA 연관관계, API 설계 |
| Redis 세션 | 2~3주차 | 세션 관리 구현 |
| 추가 기능 | 3~4주차 | 코드 품질, 예외처리 |
| 최종 정리 | 4주차 | 전체 코드 구조 |

### 홈페이지 내부 서류 (강사 작성, PDF 자동생성)

| 서류 | 시기 | 비고 |
|------|------|------|
| PBL 프로젝트 계획서 | 프로젝트 시작 전 | 이미 완료 |
| 관찰일지 (주차별) | 주 1회 입력 | 학생별 점수 입력 → PDF |
| 프로젝트 평가표 (중간/최종) | 발표 후 입력 | 6영역 100점 → PDF |
| 멘토링일지 (4회) | 멘토링 후 입력 | Q&A + 피드백 → PDF |
| 채점지/총평/체크리스트 | 평가 후 | 자동생성 |
