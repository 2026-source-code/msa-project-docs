## 코드리뷰 규칙

### 진행 방식

1. 기능 구현이 끝나면 **GitHub에서 PR(Pull Request)을 생성**한다
2. PR 제목에 `[기능명] 구현 내용 요약`을 작성한다
3. 팀원 최소 **1명 이상**이 코드를 리뷰한다
4. 리뷰어는 아래 체크리스트 기준으로 리뷰하고, **이 노션 문서에 기록**한다
5. 리뷰가 완료되면 PR을 **merge**한다

### GitHub Permalink 사용법

> 코드의 특정 라인을 링크로 남기는 방법입니다.
> 캡처 대신 링크를 사용하면 항상 해당 코드로 바로 이동할 수 있습니다.

1. GitHub에서 해당 파일을 연다
2. 리뷰할 **라인 번호를 클릭**한다 (여러 줄: Shift+클릭)
3. 좌측에 나타나는 **`...` 메뉴** → **`Copy permalink`** 클릭
4. 복사된 링크를 아래 리뷰 기록에 붙여넣는다

예시 링크: `https://github.com/팀/repo/blob/abc123/src/main/java/com/example/UserService.java#L15-L28`

### 리뷰 체크리스트

리뷰어는 아래 항목을 기준으로 코드를 확인합니다.

- [ ] **동작 확인**: 기능이 의도대로 동작하는가?
- [ ] **네이밍**: 변수명, 메서드명이 의미를 잘 전달하는가?
- [ ] **구조**: 한 메서드가 너무 길지 않은가? 역할이 분리되어 있는가?
- [ ] **예외 처리**: 예외 상황이 적절히 처리되어 있는가?
- [ ] **중복 코드**: 반복되는 코드가 있는가?
- [ ] **JPA 쿼리**: N+1 문제 등 비효율적인 쿼리가 없는가?
