### 1-1. 삼성 SSAFY (삼성 청년 SW AI 아카데미)

**과정 개요**
- 1년(약 10개월), 매일 8시간, 총 1,600시간
- 1학기(5개월): 기본과정 (Web, Embedded, Mobile, Algorithm)
- 2학기(5개월): 심화과정 (프로젝트 기반 자기주도형 학습)

**프로젝트 운영 구조**

| 단계 | 기간 | 내용 |
|-----|------|------|
| 1학기 관통 프로젝트 | 학기 말 | 2인 1조 통합 프로젝트 |
| 공통 프로젝트 | 7주 | 팀 프로젝트 (공통 기술스택) |
| 특화 프로젝트 | 7주 | 도메인별 심화 프로젝트 |
| 자율 프로젝트 | 7주 | 완전 자유주제 프로젝트 |

**강사가 작성하는 서류**
- 프로젝트 컨설팅 일지
- 프로젝트 평가 결과서

**학생에게 받는 산출물**

| 산출물 | 설명 |
|-------|------|
| 기획서 | 서비스 기획 및 기능 명세 |
| 기능 명세서 | 상세 기능 목록 + 우선순위 |
| 와이어프레임 | UI/UX 화면 설계 |
| ERD | 데이터베이스 설계도 |
| 시퀀스 다이어그램 | 기능 흐름 설계 |
| API Docs | REST API 문서 |
| 서비스 아키텍처 문서 | 시스템 구조도 |
| 포팅 매뉴얼 | 배포/환경설정 가이드 |
| README | 프로젝트 소개 문서 |
| UCC 영상 | 프로젝트 소개 영상 제작 |

**프로젝트 관리 툴**

| 툴 | 용도 |
|---|------|
| **GitLab** (lab.ssafy.com) | 코드 버전관리 (SSAFY 전용) |
| **Jira** | 이슈/스프린트 관리 |
| **Notion** | 문서 협업, 회의록 |
| **Mattermost** | 팀 커뮤니케이션 (Slack 대안) |

**평가 방법**
- 1학기: IM(Initial Modeling) 시험 통과 필수 (미통과 시 퇴소)
- 2학기: 프로젝트 컨설턴트의 발표 평가 + 결과물 심사
- 우수 프로젝트 수상 제도 (최우수상 등)

**멘토링**
- **프로젝트 컨설턴트**: 프로젝트 전반 구조, 진행 방향 상담
- **실습 코치**: 기술적 오류 해결 보조

**특징적 제도**
- UCC 영상 제작 → 프로젝트를 영상으로 소개
- Job Fair → 1학기/2학기 사이, 수료 후 기업 매칭 이벤트
- 팀별 AWS Lightsail 서버($80 스펙) + SSAFY 전용 도메인 지급

### 1-2. 네이버 부스트캠프 (Boostcamp)

**과정 개요**
- 약 8개월 풀타임, 전액 무료
- 챌린지 과정(약 1개월) → 멤버십 과정(약 4~5개월)

**프로젝트 운영 구조**

| 단계 | 내용 |
|-----|------|
| 학습 스프린트 | 분야별 도메인 지식과 기술 학습 |
| 그룹 프로젝트 | 6명 내외 팀, 사용자 고려한 서비스 개발 |

**학생에게 받는 산출물**
- 일일/주간 학습 정리 문서
- GitHub 저장소 (부스트캠프 Organization에 등록)
- 프로젝트 발표 자료

**프로젝트 관리 툴**

| 툴 | 용도 |
|---|------|
| **GitHub** | 코드 관리, 프로젝트 관리 |
| 온라인 회의 플랫폼 | 피어세션 진행 |

**평가 방법**
- 미션 기반 자체 평가
- 동료 학습(피어세션) 상호 피드백
- 그룹 프로젝트 결과 발표

**멘토링**

| 방식 | 내용 |
|-----|------|
| **피어세션** | 매일 1.5시간, 동료 학습 시간 |
| **마스터클래스** | 현업 전문가 강의 |
| **오피스아워** | 질의응답 시간 |
| 팀별 조교 | 강의 관련 질문 즉시 지원 |

**특징적 제도**
- 94명의 멘토/마스터, 85개 기업 파트너
- 챌린지에서 멤버십으로의 "선발형 학습" 구조
- 대규모 커뮤니티 학습 모델

### 1-3. 우아한테크코스 (Woowacourse)

**과정 개요**
- 약 10개월, 우아한형제들(배달의민족) 운영
- 레벨 1~4 구조

**프로젝트 운영 구조**

| 레벨 | 기간 | 내용 |
|-----|------|------|
| 레벨 1~2 | 각 7주 | 개인 미션 + 페어 프로그래밍 (2~3인) |
| 레벨 3 | 7주 | 팀 프로젝트 (서비스 기획~배포) |
| 레벨 4 | 7주 | 프로젝트 성능 개선, 레거시 리팩토링 |

**학생에게 받는 산출물**

| 산출물 | 설명 |
|-------|------|
| **글쓰기 미션** | 레벨별 1편 작성 (예: "우테코 한 달 생활기"), 교육생 간 피드백으로 완성 |
| **테코톡** | 학습한 기술 주제를 10분간 발표, **유튜브에 공개** |
| **레벨로그** | 레벨 인터뷰 대비 학습 내용 정리 문서 |
| PR 기반 미션 제출 | GitHub PR로 미션 제출 |

**프로젝트 관리 툴**

| 툴 | 용도 |
|---|------|
| **GitHub** (PR 기반) | 코드리뷰, 미션 제출 |
| 우테코 전용 Organization | woowacourse, woowacourse-teams |
| **CoReA** | 코드리뷰 매칭 플랫폼 (자체 개발) |

**평가 방법**

| 평가 방식 | 설명 |
|----------|------|
| **코드리뷰** | 외부 현직 개발자 리뷰어가 PR에 코드리뷰, merge되면 미션 완료 |
| **레벨 인터뷰** | 레벨로그 기반 모의 면접, 1명 인터뷰이 + 3명 크루 + 코치 |
| **데모데이** | 격주 프로젝트 발표, 외부 공개 |

**특징적 제도**
- **페어 프로그래밍**: 노트북 1대, 드라이버/내비게이터 역할 교대
- **테코톡 → 유튜브 공개**: 학습 내용이 포트폴리오가 됨
- **글쓰기 미션**: 기술 글쓰기 역량 체계적 훈련
- **레벨 인터뷰**: 실제 면접 환경에서 학습 점검
- **데모데이**: 격주로 외부 공개 발표

### 1-4. 42서울 (42 Seoul)

**과정 개요**
- 프랑스 Ecole 42 모델 기반
- **교수/강사 없음**, 100% 동료 학습 (Peer-to-Peer Learning)
- 무료, 자기 주도형 커리큘럼

**프로젝트 운영 구조**
- 전 과정이 프로젝트(과제) 기반
- **Piscine(피신)**: 4주간의 선발 과정, 매일 과제 수행
- 본과정: Inner Circle → Outer Circle 프로젝트 순차 진행

**학생에게 받는 산출물**
- 프로젝트별 Subject(과제 명세서) PDF에 따른 구현물
- Evaluation Sheet에 따른 평가 대응

**프로젝트 관리 툴**

| 툴 | 용도 |
|---|------|
| **Vogsphere** | 42 전용 Git 저장소 시스템 (SSH 기반) |
| **Moulinette** | 자동 테스트/채점 시스템 |
| **Norminette** | 코드 스타일 자동 검사 도구 (42 Norm 준수) |

**평가 방법**

| 방식 | 설명 |
|-----|------|
| **피어 평가 (Correction)** | Evaluation Sheet 기반, 동료가 코드 검토 + 구두 질문 |
| **Correction Point** | 평가 요청 시 2포인트 소비, 남을 평가하면 포인트 획득 |
| **Moulinette 자동 검증** | 피어 평가 통과 후 자동 테스트로 최종 검증 |
| **XP/레벨 시스템** | 과제 완료 시 XP 획득, 최대 레벨 21+ |

**특징적 제도**
- **블랙홀 시스템**: 기한 내 미완료 시 자격 박탈, 완료 시 기한 연장
- **게이미피케이션**: XP, 레벨, Achievement 등 게임 요소로 학습 동기 부여
- **42 Norm**: 엄격한 코딩 스타일 규칙 (변수명, 함수 길이, 들여쓰기 등)
- **24시간 개방 캠퍼스**
- **Voxotron**: 도움을 많이 준 동료에게 투표하는 시스템

### 1-5. 코드스테이츠 (Code States)

**과정 개요**
- SEB(Software Engineering Bootcamp) 약 6개월
- 섹션별 학습 후 프로젝트 진행

**프로젝트 운영 구조**

| 단계 | 기간 | 내용 |
|-----|------|------|
| Pre 프로젝트 | 약 2주 | Stack Overflow 클로닝 등 협업 경험 |
| Main 프로젝트 | 약 4주 | 자유 주제, 팀 단위 풀스택 서비스 개발 |

- 프론트엔드 + 백엔드 혼합 4인 팀 구성

**학생에게 받는 산출물**

| 산출물 | 설명 |
|-------|------|
| **TIL (Today I Learned)** | 일일 학습 기록 |
| **회고록** | 블로그 형태 프로젝트 회고 |
| Git Flow 기반 형상관리 | PR 컨벤션, 커밋 룰 문서 |
| 칸반보드 | 이슈 관리 기록 |

**프로젝트 관리 툴**

| 툴 | 용도 |
|---|------|

**평가 방법**
- 데일리 탐구 과제: 동기 간 **페어 피드백**

- 프로젝트 결과물 + README 평가

**특징적 제도**

- 스프린트 기반 학습: 7주간 하나의 프로덕트를 깊게 분석

### 2-1. Makers Academy (영국 런던)

**과정 개요**
- 16주 부트캠프 (월~금 09:00~17:00)
- Course Report 선정 유럽 상위 27개 부트캠프

**프로젝트 운영**
- 그룹 프로젝트 + 페어 프로젝트
- 자기 주도 학습(Self-directed learning) 강조

**평가 방법**
- **공식 평가/시험 없음**
- **외부 코드리뷰**: 외부 평가자가 코딩 과정을 관찰하고 실행 가능한 피드백 제공
- 프로젝트 결과물 중심 역량 검증

**특징적 제도**
- **EQ(감성지능) 커리큘럼**: 팀워크, 피드백, 건강 등 소프트스킬 체계적 훈련
  - 일일 체크인, 명상, 요가 세션 포함
- **페어 프로그래밍**: 첫날부터 매일, 매일 새로운 파트너 배정
- **데일리 스탠드업/레트로**: 애자일 방식 일상화
- "빠르게 새로운 것을 배우는 방법"을 가르침

### 2-2. Flatiron School (미국)

**과정 개요**
- 소프트웨어 엔지니어링, 데이터사이언스 등 Phase 1~5 단계별 교육

**프로젝트 운영**
- 각 Phase 완료 시 프로젝트 수행
- **Capstone Project**: 최종 독립 풀스택 애플리케이션

**학생에게 받는 산출물**

| 산출물 | 기준 |
|-------|------|
| **기술 블로그** | 각 Phase별 최소 1편, **750단어 이상** (3~5분 읽기 분량) 필수 |
